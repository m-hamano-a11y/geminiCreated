社内導入に向けた調査報告

自律型AI「Claude Code」
安全な社内導入ガイド

結論（エグゼクティブサマリー）

開発生産性を飛躍的に向上させるツールですが、個人向けプランの業務利用は情報漏洩リスク（AI学習へのデータ流用）を伴います。 安全な導入には、商用・エンタープライズ向けプランの契約と厳格なセキュリティ統制（ローカル端末管理・権限最小化）が不可欠です。

1. プラン選定とデータプライバシー

業務利用禁止

Free / Pro / Max プラン

個人アカウントでの利用（シャドーIT）は厳禁です。

  • 入力データがデフォルトでAIモデル学習に利用される
  • データが最長5年間保持される
推奨プラン

API / Team / Enterprise / AWS

安全な業務利用のための必須条件となります。

  • 入力データがモデル学習に一切使用されない
  • 標準保持30日。
    Enterpriseはゼロデータリテンション（ZDR）対応

2. セキュリティの懸念と必須対策

ローカル端末のデータ保護

平文保存対策

ZDR設定時でも .claude/projects/ に履歴が平文保存されます。端末暗号化(BitLocker等)とMDM/EDR導入が必須です。

アクセス権限と操作制限

ファイル制御

機密情報(.env等)を .claudeignore で除外するルールの全社標準化。人間による承認スキップ機能の利用を禁止します。

隔離環境での実行

コンテナ化

自律的なファイル操作による予期せぬ事故を防ぐため、Dockerコンテナ内（Dev Containers等）での隔離実行を推奨します。
